## Задание

### Основные требования

#### 1. Доработка класса работы с БД
Добавить в класс `StudentManager` методы:
- `update_student(student_id, update_data)` - обновление данных студента
- `delete_student(student_id)` - удаление студента
- `get_student_by_id(student_id)` - получение студента по ID

#### 2. Pydantic схемы
Создать схемы для валидации данных:
- `StudentCreate` - для создания студента
- `StudentUpdate` - для обновления студента  
- `StudentResponse` - для ответа API

#### 3. FastAPI эндпойнты
Реализовать следующие endpoints:

| Метод | URL | Описание |
|-------|-----|-----------|
| GET | `/students` | Получить всех студентов |
| GET | `/students/{id}` | Получить студента по ID |
| POST | `/students` | Создать нового студента |
| PUT | `/students/{id}` | Обновить данные студента |
| DELETE | `/students/{id}` | Удалить студента |

#### 4. Дополнительные эндпойнты
Сохранить функциональность из предыдущего ДЗ:
- `GET /students/faculty/{faculty_name}` - студенты по факультету
- `GET /courses/unique` - уникальные курсы
- `GET /faculty/{faculty_name}/average-grade` - средний балл по факультету
- `GET /courses/{course}/low-grades` - студенты курса с оценкой ниже 30
